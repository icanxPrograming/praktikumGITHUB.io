Panduan Dasar Git & GitHub

Belajar sistem version control dengan mudah dan terstruktur.

1. Apa itu Git?

Git adalah sistem version control yang digunakan untuk mencatat perubahan kode, memudahkan kolaborasi tim, dan mengelola riwayat pengembangan project. Dengan Git, kita bisa melacak perubahan, kembali ke versi sebelumnya, dan bekerja secara paralel tanpa takut kehilangan progress.

2. Apa itu GitHub?

GitHub adalah platform hosting repository berbasis Git. Dengan GitHub, kamu bisa menyimpan project-mu di cloud, berkolaborasi dengan programmer lain, melakukan pull request, review code, dan manajemen issue.

3. Perbedaan Git dan GitHub

Git adalah tools version control berbasis terminal/command line untuk menyimpan riwayat perubahan kode di lokal. Sedangkan GitHub adalah layanan cloud untuk menyimpan repository Git secara online, serta mendukung kolaborasi tim.

4. Cara Install Git

Cara Install: Git bisa di-install di berbagai sistem operasi seperti Windows, macOS, dan Linux. Untuk pengguna Windows, kamu bisa mengunduh Git melalui situs resmi di git-scm.com. Setelah proses unduhan selesai, lakukan instalasi seperti software biasa dengan pengaturan default.

Setelah Git ter-install, pastikan Git sudah bisa digunakan dengan menjalankan perintah git --version di Terminal atau Command Prompt. Jika muncul informasi versi Git, artinya instalasi berhasil.

5. Konfigurasi Git Pertama Kali

Konfigurasi Git: Setelah menginstal Git, langkah penting selanjutnya adalah mengatur username dan email yang akan dicatat di setiap commit. Jalankan perintah berikut di terminal:

git config --global user.name "Nama Kamu"
git config --global user.email "emailkamu@gmail.com"

CATATAN: Pastikan menggunakan email yang terdaftar di github.

6. Inisialisasi File dengan Git

Membuat Repository Baru dengan Git: Untuk memulai project baru dengan Git, buka terminal dan arahkan ke folder project kamu, lalu jalankan:

git init

Perintah ini akan membuat folder tersembunyi bernama .git yang berfungsi sebagai database untuk menyimpan seluruh riwayat perubahan project.

Membuat Repository Baru di GitHub: Setelah repository lokal dibuat menggunakan git init, kamu juga perlu membuat repository di GitHub agar project-mu bisa disimpan secara online.

Masuk ke situs GitHub lalu klik tombol "New Repository". Setelah itu, isi nama repository, deskripsi (opsional), lalu pilih visibilitas antara Public atau Private.

Jika sudah, klik tombol Create repository. GitHub akan menampilkan beberapa instruksi untuk menghubungkan repository lokal dengan repository GitHub.

7. Menambahkan File ke Git

Menambahkan File: Setelah membuat repository, kamu bisa menambahkan file untuk dipantau oleh Git dengan perintah:

git add nama_file atau untuk menambahkan semua file sekaligus: git add .

8. Membuat Commit

Membuat Commit: Setelah file ditambahkan, langkah selanjutnya adalah menyimpan perubahan ke repository lokal menggunakan perintah:

git commit -m "Pesan commit yang jelas"

Commit digunakan untuk mencatat snapshot project-mu saat ini, lengkap dengan pesan sebagai dokumentasi perubahan.

9. Menghubungkan Git ke Github

Menghubungkan Git ke Repository GitHub: Agar project bisa dipush ke GitHub, kamu perlu menghubungkan repository lokal ke repository remote dengan perintah:

git remote add origin https://github.com/username/nama-repo.git

Pastikan alamat URL di atas adalah link repository kamu di GitHub.

10. Push Project ke Github

Push ke GitHub: Untuk mengirimkan perubahan dari local ke repository GitHub, gunakan perintah:

git push -u origin master atau jika branch-mu bernama main: git push -u origin main

Jika berhasil, maka file kamu akan tampil di GitHub dan siap untuk dilihat oleh orang lain atau tim.
